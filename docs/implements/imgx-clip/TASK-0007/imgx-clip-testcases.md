### TC-001: 画像読み込みIPC呼び出しで画像データが取得される

- **テスト名**: 画像読み込みIPC呼び出しで画像データが取得される
  - **何をテストするか**: `handleLoadImage`がTauri `invoke('load_image')` を呼び出し、返却された`ImageMetadata`を`imageData`状態に設定する
  - **期待される動作**: `invoke`が正しい引数で呼び出され、戻り値がstateに保存される
- **入力値**: `path = "C:\\images\\sample.png"`
  - **入力データの意味**: 有効な画像ファイルパスを代表する典型的な入力値
- **期待される結果**: `imageData`が `{ base64: "...", width: 100, height: 200, format: "png", originalPath: "C:\\images\\sample.png" }` に設定される
  - **期待結果の理由**: `invoke`の戻り値にoriginalPathを追加してstateに保存するのがApp.tsxの実装仕様
- **テストの目的**: IPC連携の基本フロー（フロントエンド→Rust→フロントエンド）が正しく動作すること
  - **確認ポイント**: `invoke`の呼び出し引数、state更新、ローディング状態の遷移
- 🔵 要件定義のIPC入出力仕様・App.tsx実装コードに基づく

  - **実際の使用場面**: エラー発生→再試行成功のフロー
- **期待される結果**: `error = null`時は`div.error-message`が非表示、`error`有り時は表示
  - **境界での正確性**: `null`, `undefined`, 空文字での動作が一貫
  - **一貫した動作**: 表示/非表示の切り替えが即座に反映

  - **堅牢性の確認**: エラー状態のライフサイクル全体の検証
