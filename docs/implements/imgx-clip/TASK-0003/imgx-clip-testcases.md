- **テスト名**: top_yとbottom_yが等しい場合にバリデーションエラーが返ること
  - **エラーケースの概要**: `top_y == bottom_y` の場合、クロップ高さが0になるため不正
  - **エラー処理の重要性**: 0高さの画像は生成できないため、事前バリデーションで防止する必要がある
- **入力値**: `top_y` = 50、`bottom_y` = 50（等しい）
  - **不正な理由**: `top_y >= bottom_y` の条件に該当し、有効なクロップ範囲にならない
  - **実際の発生シナリオ**: ユーザーがドラッグ操作でY範囲を指定せずにクリックのみした場合
- **期待される結果**: `Err("上端Y座標は下端Y座標より小さい必要があります")`
  - **エラーメッセージの内容**: 日本語で原因が明確に伝わるメッセージ
  - **システムの安全性**: 画像処理に進まず即座にエラー返却。ファイルI/O不要
- **テストの目的**: Y座標バリデーション（等値ケース）の動作確認
  - **品質保証の観点**: 不正な入力に対する堅牢性を保証
