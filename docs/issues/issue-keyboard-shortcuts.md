# Issue: キーボードショートカットの追加

## 概要

主要操作にキーボードショートカットを割り当て、マウスに頼らない効率的な操作を可能にする。

## 背景・動機

画像編集ツールではキーボードショートカットが標準的に期待される。特に保存・開く・Undoなどの頻繁な操作はショートカットがあるだけで作業効率が大幅に向上する。

## 要件

### 必須ショートカット

| ショートカット | 操作 |
|---------------|------|
| `Ctrl+O` | ファイルを開く |
| `Ctrl+S` | 保存 |
| `Ctrl+Shift+S` | 別名で保存 |
| `Ctrl+Z` | Undo（Undo/Redo機能実装後） |
| `Ctrl+Y` / `Ctrl+Shift+Z` | Redo（Undo/Redo機能実装後） |

### 任意ショートカット

| ショートカット | 操作 |
|---------------|------|
| `↑` / `↓` | 選択中のラインを1px移動 |
| `Shift+↑` / `Shift+↓` | 選択中のラインを10px移動 |
| `←` / `→` | fillRightXラインを1px移動 |
| `Shift+←` / `Shift+→` | fillRightXラインを10px移動 |
| `Tab` | 次のラインを選択 |
| `Escape` | ライン選択解除 |

## 技術的な考慮事項

- `useEffect`でグローバルな`keydown`イベントリスナーを登録
- テキスト入力中（将来の数値入力フィールドなど）はショートカットを無効化
- ブラウザデフォルト動作の`preventDefault()`が必要なキー（Ctrl+Sなど）に注意
- 矢印キーによるライン移動には「選択中のライン」の概念が必要 → クリックでライン選択状態を管理

## 影響範囲

- `src/App.tsx` または新規 `src/hooks/useKeyboardShortcuts.ts` — ショートカットハンドラ
- `src/components/ImageCanvas.tsx` — ライン選択状態の管理（矢印キー移動対応時）
- `src/components/Toolbar.tsx` — ボタンにショートカットヒント表示（tooltip等）
