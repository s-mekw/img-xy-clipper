# Issue: ドラッグ&ドロップでファイルを開く

## 概要

画像ファイルをアプリウィンドウにドラッグ&ドロップすることで、ファイルダイアログを経由せずに画像を読み込めるようにする。

## 背景・動機

エクスプローラーから画像を直接ドロップして開くのは、デスクトップアプリとして自然な操作。特にフォルダ内の画像を次々と編集する場合、ファイルダイアログを毎回開くよりドロップの方が速い。

## 要件

### 必須

- ウィンドウ全体をドロップターゲットにする
- PNG/JPEG（.png, .jpg, .jpeg）のみ受け付ける
- ドラッグ中のビジュアルフィードバック（例: 枠線のハイライト、オーバーレイ表示）
- ドロップ後は既存の`load_image`コマンドを利用して画像を読み込む
- 対応外ファイルをドロップした場合はエラーメッセージを表示

### 任意

- 複数ファイルをドロップした場合は最初の1つのみ読み込む（または警告表示）
- 画像未読み込み時は「ここにファイルをドロップ」のプレースホルダーを表示

## 技術的な考慮事項

- Tauri v2の`onDragDropEvent`またはWebViewの`dragover`/`drop`イベントを利用
  - Tauri v2では`@tauri-apps/api/webview`の`onDragDropEvent`が利用可能
  - ファイルパスを直接取得できるため、Webの`FileReader`より適切
- ドロップで受け取ったファイルパスを既存の`load_image` IPCコマンドに渡す
- `dragover`イベントで`preventDefault()`を忘れるとドロップが動作しない点に注意
- 既に画像を読み込み済みの場合、確認なしで新しい画像に切り替えるか確認ダイアログを出すか検討

## 影響範囲

- `src/App.tsx` — ドロップイベントハンドラ、ドラッグ中の状態管理
- `src/styles/index.css` — ドラッグ中のビジュアルフィードバック用スタイル
- `src-tauri/capabilities/default.json` — ドラッグ&ドロップ関連のパーミッション追加（必要な場合）
