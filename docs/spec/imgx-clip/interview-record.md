# imgX-Clip ヒアリング記録

**作成日**: 2026-03-11
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

新規プロジェクトの要件を明確化するため、機能・技術スタック・操作方法についてヒアリングを実施しました。

## 質問と回答

### Q1: クリップ方向の確認

**カテゴリ**: 未定義部分詳細化
**背景**: 入力ファイルには「X軸の範囲を切り取る」と記載されていたが、スクリーンショットを見ると縦方向（Y軸）の切り取りに見えたため確認が必要だった

**回答**: Y軸範囲指定で切り取り

**信頼性への影響**:
- この回答により、REQ-002の信頼性レベルが 🔴 → 🔵 に向上
- クリップ方向がY軸であることが確定

---

### Q2: 技術スタックの選定

**カテゴリ**: 追加要件
**背景**: デスクトップアプリの技術スタックが未定だったため、ユーザーの希望を確認

**回答**: 軽量がよく、範囲はユーザーが動的に決めたい。C# + WPFも検討したが、ドラッグ操作の体験比較を経て **Tauri + React** に決定

**信頼性への影響**:
- この回答により、REQ-401（技術スタック）の信頼性レベルが 🔵 に確定
- ドラッグUIの操作感を重視した選定

---

### Q3: クリップ範囲の指定方法

**カテゴリ**: 未定義部分詳細化
**背景**: 範囲指定を数値入力にするかGUIドラッグにするか確認が必要だった

**回答**: GUIでドラッグ

**信頼性への影響**:
- この回答により、REQ-002の信頼性レベルが 🔵 に確定

---

### Q4: 保存形式

**カテゴリ**: 未定義部分詳細化
**背景**: 出力画像の形式を確認

**回答**: 入力と同じ形式

**信頼性への影響**:
- この回答により、REQ-003の信頼性レベルが 🔵 に確定

---

### Q5: リアルタイム拡大プレビュー

**カテゴリ**: 追加要件
**背景**: ユーザーから「ドラッグ中にその領域だけ拡大表示できるか？動的に」という要望があった

**回答**: リアルタイム拡大プレビュー機能を要件に追加

**信頼性への影響**:
- 新規要件 REQ-004 を追加（信頼性レベル: 🔵）

---

## ヒアリング結果サマリー

### 確認できた事項
- クリップ方向はY軸（縦方向）
- 技術スタックはTauri + React (TypeScript)
- 範囲指定はGUIドラッグ操作
- 保存形式は入力と同じ形式
- ドラッグ中のリアルタイム拡大プレビューが必要

### 追加/変更要件
- REQ-004（リアルタイム拡大プレビュー）を新規追加
- 入力ファイルの「X軸」を「Y軸」に修正

### 残課題
- 対応する画像形式の詳細（PNG/JPG以外の対応範囲）
- 拡大プレビューの表示位置・サイズの詳細仕様

**ヒアリング前**:
- 🔵 青信号: 0件
- 🟡 黄信号: 1件
- 🔴 赤信号: 4件

**ヒアリング後**:
- 🔵 青信号: 5件 (+5)
- 🟡 黄信号: 2件 (+1)
- 🔴 赤信号: 0件 (-4)
