# imgX-Clip 要件定義書（軽量版）

## 概要

画像を入力として、Y軸方向の特定範囲を切り取った画像を生成・保存するデスクトップアプリケーション。
ユーザーはGUIでドラッグ操作により切り取り範囲を動的に指定でき、選択範囲のリアルタイム拡大プレビューを確認しながら操作できる。

## 技術スタック

- **フレームワーク**: Tauri v2 + React (TypeScript)
- **画像処理**: Rust側（image crate）またはCanvas API
- **UI**: React + HTML Canvas（ドラッグ操作・プレビュー）
- **ビルド**: 単体exe配布（Windows）

## 主要機能要件

**【信頼性レベル凡例】**:
- 🔵 **青信号**: ユーザヒアリングを参考にした確実な要件
- 🟡 **黄信号**: ユーザヒアリングから妥当な推測による要件
- 🔴 **赤信号**: ヒアリングにない推測による要件

### 必須機能（Must Have）

- REQ-001: ユーザーは画像ファイルを読み込めなければならない（PNG/JPG等の一般的な画像形式に対応） 🔵 *ユーザヒアリングより*
- REQ-002: ユーザーはGUI上で画像をドラッグ操作してY軸方向の切り取り範囲を動的に指定できなければならない 🔵 *ユーザヒアリングより*
- REQ-003: システムは指定されたY軸範囲で画像を切り取り、結果を入力と同じ形式で保存しなければならない 🔵 *ユーザヒアリングより*
- REQ-004: システムはドラッグ操作中に選択範囲をリアルタイムで拡大プレビュー表示しなければならない 🔵 *ユーザヒアリングより*

### 基本的な制約

- REQ-401: アプリケーションはTauri v2 + React (TypeScript)で構築しなければならない 🔵 *ユーザヒアリングより*
- REQ-402: アプリケーションはWindows上で動作しなければならない 🔵 *環境情報より*
- REQ-403: ビルド後は軽量な単体実行ファイルとして配布可能でなければならない 🟡 *Tauri選定理由から妥当な推測*

## 簡易ユーザーストーリー

### ストーリー1: 画像の読み込みとクリップ

**私は** ユーザー **として**
**画像を読み込み、Y軸方向の不要な範囲をドラッグで選択して切り取りたい**
**そうすることで** 必要な部分だけの画像を素早く作成できる

**関連要件**: REQ-001, REQ-002, REQ-003

### ストーリー2: リアルタイムプレビューで確認

**私は** ユーザー **として**
**ドラッグ操作中にクリップ後の画像をリアルタイムで拡大プレビューしたい**
**そうすることで** 正確な範囲指定を視覚的に確認しながら操作できる

**関連要件**: REQ-004

## 基本的な受け入れ基準

### REQ-001: 画像読み込み

**Given**: アプリが起動している
**When**: ユーザーが画像ファイルを選択する（ファイルダイアログまたはD&D）
**Then**: 画像がCanvas上に表示される

**テストケース**:
- [ ] 正常系: PNG/JPG画像を読み込み表示できる
- [ ] 主要な異常系: 非対応形式の場合エラーメッセージを表示する

### REQ-002: Y軸範囲のドラッグ選択

**Given**: 画像が読み込まれている
**When**: ユーザーが画像上でY軸方向にドラッグする
**Then**: 選択範囲がオーバーレイで視覚的に表示される

**テストケース**:
- [ ] 正常系: ドラッグでY軸範囲を選択できる
- [ ] 主要な異常系: 画像範囲外へのドラッグは境界でクランプされる

### REQ-003: 画像の切り取りと保存

**Given**: Y軸範囲が選択されている
**When**: ユーザーが保存操作を実行する
**Then**: 選択範囲で切り取られた画像が入力と同じ形式で保存される

**テストケース**:
- [ ] 正常系: 切り取り後の画像が正しいサイズ・形式で保存される
- [ ] 主要な異常系: 保存先に書き込み権限がない場合エラーメッセージを表示する

### REQ-004: リアルタイム拡大プレビュー

**Given**: 画像が読み込まれている
**When**: ユーザーがドラッグ操作中
**Then**: 選択範囲の拡大プレビューがリアルタイムで表示される

**テストケース**:
- [ ] 正常系: ドラッグ中に選択範囲が拡大表示される
- [ ] 正常系: ドラッグ範囲変更に追従して拡大表示が更新される

## 最小限の非機能要件

- **パフォーマンス**: ドラッグ操作中のプレビュー更新は60fps程度を維持する 🟡 *一般的なUX基準から推測*
- **ユーザビリティ**: 画像の読み込みからクリップ・保存まで3ステップ以内で完了できる 🟡 *シンプルなアプリ設計から推測*
