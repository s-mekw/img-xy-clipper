## タスク概要

フロントエンド（React）とバックエンド（Rust）のIPC連携を結合し、アプリケーション全体のEnd-to-End動作を確認する。画像読み込み→ドラッグ操作→プレビュー→クリップ保存の一連のフローが正しく動作することを検証する。

## 完了条件

- [ ] ファイルダイアログから画像を読み込み、Canvas上に表示される
- [ ] 2本の水平線をドラッグしてY軸範囲を選択できる
- [ ] 選択中にリアルタイム拡大プレビューが表示される
- [ ] 保存ボタンで選択範囲がクリップされた画像が保存される
- [ ] エラー時にメッセージが表示される

## 主要実装項目

### IPC連携の結合 🔵

**信頼性**: 🔵 *データフロー図の全体フローより*

- Toolbar → `@tauri-apps/api` → Rust `load_image` → ImageCanvas表示
- Toolbar → `@tauri-apps/api` → Rust `clip_and_save` → 保存完了通知
- エラーハンドリングの統合（Rust側エラー → フロントエンドエラー表示）

**実装ファイル**: `src/components/Toolbar.tsx`, `src/App.tsx`

### E2E動作確認 🔵

**信頼性**: 🔵 *要件定義のユーザーストーリーより*

- ストーリー1: 画像読み込み → ドラッグ範囲指定 → クリップ保存
- ストーリー2: ドラッグ中にリアルタイムプレビュー確認
- エラーケース: 非対応形式の画像読み込み

**実装ファイル**: 手動テスト
